# Как обновить PostgREST кэш в Supabase

## Проблема

После применения миграций PostgREST может не видеть новые колонки, функции или изменения в схеме из-за кэша.

## Решение 1: Через Dashboard (рекомендуется)

1. Откройте Supabase Dashboard
2. Выберите ваш проект
3. Нажмите на **Settings** (⚙️)
4. Выберите **API**
5. Прокрутите вниз до секции **"API Settings"**
6. Нажмите кнопку **"Reload schema"** или **"Restart PostgREST"**

## Решение 2: Через SQL

Выполните в SQL Editor:

```sql
NOTIFY pgrst, 'reload schema';
```

## Решение 3: Подождите

PostgREST автоматически обновит кэш через ~10 минут.

## Проверка

После перезагрузки схемы попробуйте зарегистрироваться снова:
- http://localhost:3000/register

## Альтернатива (текущая реализация)

Код регистрации использует SQL функцию `create_tenant()` которая обходит PostgREST кэш и работает напрямую с базой данных.

Это позволяет регистрации работать даже если PostgREST кэш не обновлен.
